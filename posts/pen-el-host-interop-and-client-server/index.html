Pen.el host interop and client/server

Summary

Pen.el may now run as a server. On your host OS, you can create symlinks to the pen script with names matching the name of prompts. Running those symlinks will then run the prompt via the server and return the result.

Alternatively, you can run pen pf-my-prompt-function "first arg" "second arg" "etc." on the host machine and it will run the prompt function on the server and return the output as stdout on the host.

You can even pipe into pen or the symlinks and the stdin will be used as the first argument to the prompt.

This means it is now dead simple to integrate GPT-3 and other LMs easily into your bash pipelines and other software. Pen.el is now your management system for LMs and prompting.

You may create multiple client interfaces to Pen.el, which means you can have many prompt-engineering workflows going at once.

You may want to be working with multiple documents, for example, or have a separate client just for prompt engineering.

Using prompt functions in bash

Start the server on the host and open a separate terminal elsewhere

  • Run pen in a terminal.
  • Open a terminal

List out the prompt functions and get help

1

penl | fzf
1

penh pf-code-one-liners-from-natural-language
(pf-code-one-liners-from-natural-language &optional LANGUAGE TASK &key NO-SELECT-RESULT)

Caching is default

1

Special functions

pen-update

This updates the disk memoization.

You may use pene (or pen -e) and specify pen-update manually, or you may use penu (pen -u) which has a default pen-update.

1

pene '(car (pen-update (pf-list-of 7 "tennis players" :no-select-result t)))'
Andre Agassi
Billie Jean King
Samantha Stosur
Roger Federer
Andy Murray

You may also use prompt functions this way

1

pen pf-nlsh "arch linux" "install python package"
pacman -S python-pip

Another way to update is to prefix upd.

I’m going to deliberately request an ambiguous command “open an image”.

1

upd pen pf-nlsh "arch linux" "open an image"
rxvt -bg black -fg white -geometry 130x30 foo.png
sxiv foo.png
feh --bg-scale foo.png
gthumb foo.png
eog foo.png

You may also use the UPDATE environment variable.

1

UPDATE=y pen pf-nlsh "arch linux" "open an image"
feh filename.png
imagemagick filename.jpg
convert image.jpg image.png

Running a prompt function from the host

Firstly, start a server

1

pen

To run additional clients, just run pen again in a different terminal.

Then run a prompt function

1

pen -e '(car (pf-list-of 5 "tennis players" :no-select-result t))'
Elena Dementieva
Roger Federer
Marat Safin
Anastasia Myskina
Andre Agassi

Alternatively, make a symlink to the pen script

Like so:

1

ln -sf pen pf-list-of

Then you may run the functions as scripts on your host.

1

pf-list-of 5 "tennis players"
Elena Dementieva
Roger Federer
Marat Safin
Anastasia Myskina
Andre Agassi
